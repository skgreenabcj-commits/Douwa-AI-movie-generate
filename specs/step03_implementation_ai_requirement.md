# 02_Scenes フィールド定義・補足・例示（桃太郎ベース）

本ドキュメントは、`02_Scenes` シートにおける主要フィールドについて、  
AI に期待する役割、定義補足、入力・出力イメージ、他 STEP での活用用途を明確化するための設計補助資料である。

対象フィールド:
- `chapter`
- `scene_title`
- `scene_summary`
- `scene_goal`
- `visual_focus`
- `emotion`
- `short_use`
- `full_use`
- `est_duration_short`
- `est_duration_full`
- `difficult_words`
- `easy_rewrite`
- `qa_seed`
- `continuity_note`

前提:
- STEP_03 は後工程の正本となる scene master を定義する
- STEP_04 / STEP_05 / STEP_06 / STEP_07 / STEP_08 / STEP_09 は `02_Scenes` を基準に生成する
- scene は桃太郎のような昔話でも、後工程で script / visual / QA に転用しやすい粒度で定義する
- 例示は桃太郎をベースにしているが、定義自体は他作品にも再利用可能とする
- `short_use` と `full_use` は `Y` / `N` のみを取る利用可否フラグである

---

| field | AIに求める役割 | 主な用途 |
|---|---|---|

| scene_title | scene を一目で識別できる短い見出し | 人レビュー、script/visual参照 |
| scene_summary | その scene で何が起きるかを短く要約 | script生成、QA生成、全体確認 |
| scene_goal | その scene の役割・意味を定義 | scene設計の一貫性維持 |
| visual_focus | 映像化の主対象を明示 | Visual Bible / Image Prompt |
| emotion | scene の感情トーンを定義 | Script / VO / Music / Visual |
| short_use | short版で使うかどうかを示す | STEP_04 対象判定 |
| full_use | full版で使うかどうかを示す | STEP_05 対象判定 |

## 1. chapter

### AIに求める役割
scene が物語全体の中でどの章に属するかを示す。  
scene 単体ではなく、複数 scene を束ねる大きな構造単位として使う。

### 定義項目の補足
- chapter は「起・承・転・結」のような抽象ラベルでもよい
- あるいは「出発」「試練」「対決」「帰還」など物語依存の章名でもよい
- 同じ chapter に複数 scene が含まれてよい
- chapter は後工程で scene 群のまとまりを理解するための補助情報であり、script の最小単位ではない

### 例示（桃太郎）
- `誕生と旅立ち`
- `鬼ヶ島への道`
- `鬼との対決`
- `帰還と祝い`

### プット例
- `鬼ヶ島への道`

### 他STEPへの活用用途
- STEP_05 Full Script で章立て構成の補助
- STEP_06 Visual Bible で章ごとの色味やトーン整理
- 人レビュー時の流れ確認

## 2. scene_title

### AIに求める役割
scene を短く識別できるタイトルをつける。  
人間にも他 AI にも「この scene が何か」が即座に分かるようにする。

### 定義項目の補足
- 1行で読める短さにする
- 動作・変化・出来事が分かる表現にする
- 抽象的すぎる語は避ける
- scene_summary の見出し版と考える

### 例示（桃太郎）
- `大きな桃が流れてくる`
- `桃から男の子が生まれる`
- `きびだんごで仲間を集める`
- `鬼ヶ島で鬼と向き合う`

### プット例
- `きびだんごで仲間を集める`

### 他STEPへの活用用途
- STEP_04 / STEP_05 で script section 見出し
- STEP_06 / STEP_07 で visual prompt の参照名
- STEP_09 で QA 対象 scene の識別子補助

---

## 3. scene_summary

### AIに求める役割
その scene で起きる出来事を、1〜3文程度で簡潔に要約する。

### 定義項目の補足
- 「何が起きるか」を中心に書く
- 前後 scene を読まなくても scene の内容が分かるようにする
- script そのものではなく、構造要約にする
- 説明しすぎず、visual / script に展開できる余白を残す

### 例示（桃太郎）
- `桃太郎は鬼ヶ島へ向かう途中で、犬・猿・きじに出会う。きびだんごを分けることで仲間になり、みんなで鬼ヶ島へ進む。`

### プット例
- `桃太郎は旅の途中で犬・猿・きじに出会い、きびだんごを分けて仲間にする。鬼ヶ島へ向かう仲間がそろう場面である。`

### 他STEPへの活用用途
- STEP_04 / STEP_05 で narration 展開の元データ
- STEP_06 で scene の visual concept 抽出
- STEP_09 で QA 生成の基礎文脈

---

## 4. scene_goal

### AIに求める役割
その scene が物語全体の中で果たす役割を定義する。

### 定義項目の補足
- 「何が起きるか」ではなく「なぜこの scene が必要か」を書く
- 例: 導入、仲間集結、対立提示、山場、解決、余韻
- scene_summary と似て見えるが、goal は機能説明である
- 後工程が scene の重要度を理解するために必要

### 例示（桃太郎）
- `桃太郎が一人ではなく仲間と進む物語であることを示し、鬼退治への期待を高める。`

### プット例
- `仲間集結を通じて、鬼退治が本格的に始まることを示す。`

### 他STEPへの活用用途
- STEP_04 / STEP_05 で scene ごとの narrative emphasis 制御
- STEP_06 で visual の見せ場強弱決定
- STEP_11 QC で scene 必要性レビュー

---

## 5. visual_focus

### AIに求める役割
その scene を映像化するときに、最も強調すべき視覚対象を明示する。

### 定義項目の補足
- 人物、動作、道具、背景、構図、象徴物のうち主役を明確にする
- 1 scene に要素が多くても「最も見せたいもの」を優先して書く
- 映像・画像生成に直接つながる情報にする
- 抽象語だけでなく具体物を含めるとよい

### 例示（桃太郎）
- `桃太郎がきびだんごを差し出し、犬・猿・きじが集まる場面`
- `鬼ヶ島の門の前に立つ桃太郎たちのシルエット`

### プット例
- `桃太郎の手にあるきびだんごと、それを見つめる犬・猿・きじの関係性`

### 他STEPへの活用用途
- STEP_06 Visual Bible の scene visual definition
- STEP_07 Image Prompts の主題
- STEP_08 Edit Plan のカット重点

---

## 6. emotion

### AIに求める役割
scene 全体の感情トーンを明示する。

### 定義項目の補足
- 観客が感じるべき主感情を書く
- 1語〜短句でよい
- 複数感情がある場合も主感情を優先する
- 声・音・色・テンポに関わるので重要

### 例示（桃太郎）
- `わくわく`
- `安心`
- `緊張`
- `勇気`
- `達成感`

### プット例
- `わくわくと心強さ`

### 他STEPへの活用用途
- STEP_04 / STEP_05 の語り口調整
- STEP_06 の色設計・構図
- STEP_08 のBGM / TTS テンポ設計

## 7. short_use

### AIに求める役割
その scene を Short版の生成対象に含めるかどうかを示す利用可否フラグを返す。

### 定義項目の補足
- 取りうる値は `Y` または `N` のみとする
- `Y` は「Short版でこの scene を使用する」を意味する
- `N` は「Short版ではこの scene を使用しない」を意味する
- この項目は詳細な圧縮方法や演出方針を書く欄ではない
- Short版での表現密度や脚本化の詳細は、後続 STEP または別ルールで扱う

### 例示（桃太郎）
- 桃太郎誕生 scene → `Y`
- 仲間集結 scene → `Y`
- 長い移動経過の補足 scene → `N` の可能性あり

### プット例
- `Y`

### 他STEPへの活用用途
- STEP_04 Short Script の対象 scene 抽出
- Short版の scene 採用判定
- Short版の尺調整時の機械判定条件

---

## 8. full_use

### AIに求める役割
その scene を Full版の生成対象に含めるかどうかを示す利用可否フラグを返す。

### 定義項目の補足
- 取りうる値は `Y` または `N` のみとする
- `Y` は「Full版でこの scene を使用する」を意味する
- `N` は「Full版ではこの scene を使用しない」を意味する
- Full版は原則として scene を広く保持する想定だが、設計上不要な scene は `N` を許容する
- この項目も詳細な描写方針を書く欄ではない

### 例示（桃太郎）
- 桃太郎誕生 scene → `Y`
- 鬼との対決 scene → `Y`
- 重複説明だけの scene → `N` の可能性あり

### プット例
- `Y`

### 他STEPへの活用用途
- STEP_05 Full Script の対象 scene 抽出
- Full版の scene 採用判定
- scene ごとの script 生成要否の制御

---

## 9. est_duration_short

### AIに求める役割
short版でのその scene の想定尺を秒数で示す。

### 定義項目の補足
- 数値のみ、またはシステム指定形式に従う
- short全体時間内で収まるよう配分する
- STEP_03 の設計時点で rough estimate を持つことで後工程が安定する

### 例示（桃太郎）
- `6`
- `8`
- `10`

### プット例
- `8`

### 他STEPへの活用用途
- STEP_04 Short Script の文字量制御
- STEP_08 TTS / Subtitle / Edit Plan の秒数管理
- 全体尺チェック

---

## 10. est_duration_full

### AIに求める役割
full版でのその scene の想定尺を秒数で示す。

### 定義項目の補足
- STEP_03 は full版の scene master 設計も兼ねるため、full想定秒数は重要
- 年齢帯別 `scene_max_sec` を超えないこと
- 物語上の重要 scene ほど長くてもよいが、制約内で調整する

### 例示（桃太郎）
- `18`
- `22`
- `25`

### プット例
- `20`

### 他STEPへの活用用途
- STEP_05 Full Script 長制御
- STEP_08 Edit Plan
- scene count / duration validation

---

## 11. difficult_words

### AIに求める役割
対象年齢に対して難しい可能性がある語を抽出する。

### 定義項目の補足
- 作品固有語、古語、抽象語、難読語を含む
- 対象年齢を前提に判断する
- カンマ区切りや配列形式など、schema に合わせて統一する
- script 化の前段で言語難易度を制御する役割を持つ

### 例示（桃太郎）
- `鬼ヶ島, きびだんご, 家来, 退治`

### プット例
- `鬼ヶ島, きびだんご, 家来`

### 他STEPへの活用用途
- STEP_04 / STEP_05 の語彙調整
- STEP_08 字幕や読み上げ最適化
- STEP_09 Q&A の言い換え設計

---

## 12. easy_rewrite

### AIに求める役割
`difficult_words` に対応する、対象年齢向けのやさしい言い換え案を示す。

### 定義項目の補足
- 元語を完全に消すのではなく、補足表現として扱ってもよい
- 年齢帯によっては「残す + 簡単に説明する」が適切な場合もある
- 語彙難易度調整の候補集として使う

### 例示（桃太郎）
- `鬼ヶ島→鬼のいるしま`
- `家来→なかま`
- `退治→やっつける`

### プット例
- `鬼ヶ島→鬼のいるしま / 家来→なかま / 退治→やっつける`

### 他STEPへの活用用途
- STEP_04 / STEP_05 の script rewrite
- STEP_08 subtitle readability
- STEP_09 年齢別 QA 調整

---

## 13. qa_seed

### AIに求める役割
その scene から作りやすい問い・確認ポイントの種を作る。

### 定義項目の補足
- 質問文そのものでも、質問テーマでもよい
- 「何が起きた？」「なぜそうした？」「どう感じた？」の種になる内容が望ましい
- STEP_09 で QA を作る際のヒントとして使う

### 例示（桃太郎）
- `桃太郎はどうして犬たちと仲間になれたの？`
- `きびだんごはどんな役目をしたの？`

### プット例
- `桃太郎はどうやって仲間を増やしたの？`

### 他STEPへの活用用途
- STEP_09 Q&A Build の直接入力種
- 教育的観点での理解確認
- Review 用の読解ポイント整理

---

## 14. continuity_note

### AIに求める役割
前後 scene と自然につながるための注意点を記録する。

### 定義項目の補足
- 衣装、持ち物、位置関係、感情状態、時間帯、進行方向などの連続性を書く
- script と visual のズレ防止のために重要
- scene 単体ではなく scene 間の橋渡し情報として扱う

### 例示（桃太郎）
- `前 scene で持っていたきびだんご袋をこの scene でも保持する。犬→猿→きじの順で仲間が増える流れを崩さない。`

### プット例
- `桃太郎は旅装束のまま。きびだんご袋を持ち続け、仲間が増える順番と関係性を次 scene へ引き継ぐ。`

### 他STEPへの活用用途
- STEP_04 / STEP_05 script continuity
- STEP_06 / STEP_07 visual continuity
- STEP_08 edit continuity / subtitle consistency

### 尺の考え方
- Full版は上表の `est_duration_full` 合計で **約 480 秒想定**
- Short版は `short_use = Y` の scene を採用し、`est_duration_short` 合計で **約 240 秒想定**
- `short_use = N` の scene は、Short版では独立 scene として使わず、必要に応じて前後 scene の script 内に要約吸収する余地を持つ

### short_use 厳選の意図
Short版では以下を優先して残す。
- 物語の始まりが分かる scene
- 主人公の目的が明確になる scene
- 仲間集結が分かる scene
- 対決と勝利が分かる scene
- 安心して終われる結末 scene

Short版で省略候補になりやすいもの:
- 成長の説明だけを担う中間 scene
- 問題提示の補助 scene
- 仲間集結のうち重複度が高い scene
- 帰還途中のつなぎ scene

### 実装メモ
- `full_use` はこのサンプルでは全 row `Y` としている
- `short_use` は 240 秒版の優先度に応じて選別している
- 将来的に short 吸収ロジックを入れる場合も、scene master 自体は保持する

## 実装時の補足指針

- `scene_summary` は script ではなく構造要約として扱う
- `scene_goal` は「その scene の存在意義」を表す
- `visual_focus` は image prompt に直結できる具体性を優先する
- `short_use` / `full_use` は `Y` / `N` のみを返す利用可否フラグとして扱う
- `short_use` / `full_use` は scene の削除や統合の詳細方針を書く欄ではない
- `est_duration_short` / `est_duration_full` は後工程で秒数検証に使う
- `difficult_words` / `easy_rewrite` は対象年齢に応じた言語難易度制御のために保持する
- `qa_seed` は STEP_09 の直接入力候補とする
- `continuity_note` は後工程同期性の要であり、軽視しない
