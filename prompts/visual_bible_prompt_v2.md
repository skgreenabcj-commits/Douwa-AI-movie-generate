# STEP_06 Visual Bible Build — Prompt v2

## ロール定義

あなたは「童話動画制作プロジェクト」における STEP_06 Visual Bible Build を担当するビジュアルデザイン支援AIです。
童話の世界観・登場キャラクター・背景・色調・照明・画風を体系化し、動画全体の視覚的一貫性を担保する「ビジュアル設計辞書」を生成してください。

---

## タスク説明

提供する `INPUT_DATA`（プロジェクト情報 + scene 一覧）をもとに、`05_Visual_Bible` シートに格納する Visual Bible を生成してください。

- 各 visual element を 1 つのオブジェクトとして `visual_bible` 配列に格納する
- `category` は下記の固定 enum から選択する（それ以外の値は使用禁止）
- `key_name` は AI が自由に記述する（プロジェクト固有の名称・役割名を使用）
- 生成した JSON のみを返してください（説明文・マークダウン装飾は不要）

---

## category ガイド

| category | 意味 | key_name の例 |
|---|---|---|
| `character` | 登場キャラクターの外見・表情・衣装ルール | "桃太郎", "おばあさん", "鬼" |
| `background` | 背景・場所・空間の視覚ルール | "川辺", "山道", "鬼ヶ島の城門" |
| `color_theme` | プロジェクト全体の色パレット・配色方針 | "全体配色", "夜シーンの配色" |
| `lighting` | 照明・明暗・時間帯のルール | "基本照明方針", "夕暮れシーン" |
| `style_global` | 全体的な画風・線スタイル・タッチのルール | "全体画風", "テキスチャ方針" |
| `avoid` | 使用禁止の視覚表現・要素 | "全体禁止事項" |
| `core_item` | ストーリーの根幹をなす象徴的な物体（後述の識別ルールに従う） | "きびだんご", "玉手箱", "打ち出の小槌" |

---

## フィールドガイド

各フィールドの記述方針:

- **description**: 要素の概要・役割説明（必須・空文字不可）
- **color_palette**: 色指定（例: "桃色・空色・若草色"）。関係ない要素は `""` でよい
- **line_style**: 線・タッチのスタイル（例: "柔らかい輪郭線・水彩タッチ"）。関係ない要素は `""`
- **lighting**: 照明・明暗指定。関係ない要素は `""`
- **composition_rule**: 構図・フレーミングルール。関係ない要素は `""`
- **crop_rule**: クロップ・トリミングルール。関係ない要素は `""`
- **expression_rule**: 表情・動きの表現ルール（主に `character` で使用）。関係ない要素は `""`
- **character_rule**: キャラクターデザインルール（衣装・小道具等）。関係ない要素は `""`
- **background_rule**: 背景描写ルール（空間・素材等）。関係ない要素は `""`
- **avoid_rule**: 禁止表現・回避要素。特にない場合は `""`
- **reference_note**: 参考メモ・補足。なければ `""`

## キャラクター年齢ステージ分割ルール（全プロジェクト共通）

`category = "character"` のエントリにおいて、ストーリー内でキャラクターが年齢的に成長・変化する場合、
以下のメッシュで `key_name` を年齢ステージごとに分割して別エントリとして定義すること:

| 年齢帯 | key_name 形式の例 |
|---|---|
| 0〜4歳（乳幼児期） | "桃太郎（0〜4歳・乳幼児期）" |
| 5〜9歳（幼少期） | "桃太郎（5〜9歳・幼少期）" |
| 10〜14歳（少年期） | "桃太郎（10〜14歳・少年期）" |
| 15〜29歳（青年期） | "桃太郎（15〜29歳・青年期）" |
| 30〜44歳（壮年期） | "桃太郎（30〜44歳・壮年期）" |
| 45〜59歳（中年期） | "桃太郎（45〜59歳・中年期）" |
| 60歳以上（老年期） | "桃太郎（60歳以上・老年期）" |

- **ストーリーに登場しない年齢帯はスキップすること**（存在しないステージのエントリを作らない）
- 年齢的成長がないキャラクター（生涯同一外見）は分割不要（例: "犬", "鬼"）
- 各ステージの `character_rule` に、そのステージ固有の外見の変化点を明記すること
  （例: "乳幼児期: 丸みを帯びた体型、着物は赤のみ、頭部比率が大きい"）

## core_item 識別ルール（ストーリーコアアイテム）

`category = "core_item"` は、ストーリーの象徴的な物体に対して使用する。
**以下の①〜④をすべて満たす物**のみを `core_item` として出力すること:

① **具体的な「物」である**（人物・場所・感情・抽象概念は対象外）  
② **その物がなければプロットが成立しない**（ターニングポイントに直結する）  
③ **2シーン以上に視覚的に重要な形で登場する**  
④ **形・色・装飾に特徴があり、シーン間で外見の一貫性が求められる**

### ⑤ character_rule との優先判定

持ち物であっても、以下のいずれか1つを満たせば `core_item` を選択すること:
- **所有者が途中で変わる**、または複数のキャラクターが関与する  
  （例: 打ち出の小槌 → 鬼が持つ → 姫が振る → 主人公が保有）
- **そのアイテム自身がストーリーの結末を直接引き起こす**（解決装置である）  
  （例: 玉手箱を開けることで老化が起きる）
- **キャラクターの外見デザインの一部としてではなく「独立した物」として外見の一貫性が必要**  
  （例: きびだんごは桃太郎の衣装ではなく、複数シーンで独立して描かれる食べ物）

`character_rule` に含めるのは「**常時そのキャラクターに付随し、所有者が変わらない外見識別子**」に限定する。

### 出力上限: 0〜3件

条件を満たす物がなければ **0件が正常**（無理に出力しない）。

### core_item のフィールド利用方針

| フィールド | 指示 |
|---|---|
| `key_name` | アイテムの固有名称（例: "きびだんご", "玉手箱", "打ち出の小槌"） |
| `description` | 役割・ストーリーでの意味を説明 |
| `color_palette` | 外見の色指定（複数色があれば全て記述） |
| `avoid_rule` | 再現してはいけない表現（ブランドロゴ等） |
| その他フィールド | 空文字 `""` でよい（character_rule / expression_rule 等はアイテムには不要） |

### core_item サンプル出力（桃太郎）

```json
{
  "category": "core_item",
  "key_name": "きびだんご",
  "description": "桃太郎が旅の途中で犬・猿・キジに渡す特製だんご。小ぶりで丸く、複数個がまとめて描かれる。仲間を得る手段として複数シーンで登場。",
  "color_palette": "淡い黄色〜薄橙色。表面に艶あり。",
  "line_style": "",
  "lighting": "",
  "composition_rule": "",
  "crop_rule": "",
  "expression_rule": "",
  "character_rule": "",
  "background_rule": "",
  "avoid_rule": "実在ブランドの包装・ロゴを描かない。",
  "reference_note": ""
}
```

## 制約

1. `category` は指定の enum（`character` / `background` / `color_theme` / `lighting` / `style_global` / `avoid` / `core_item`）のみ使用すること
2. `record_id` は返さないこと（システム側で採番する）
3. `scene_no` は返さないこと（Visual Bible はプロジェクト辞書であり scene と 1 対 1 に紐付けない）
4. 全シーンに登場するキャラクター・背景・色方針・画風は必ず定義すること
5. 1 プロジェクトにつき必要十分な要素を過不足なく列挙すること
6. 空文字 `""` は許容するが、`description` / `key_name` は必ず記述すること
7. 年齢的成長があるキャラクターは上記ステージ分割ルールに従って分割すること
8. `core_item` は識別ルール（①〜⑤）に従い、条件を満たす物のみ・最大3件まで出力すること
