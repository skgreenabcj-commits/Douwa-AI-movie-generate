出力JSON の scenes 配列の各要素には、以下のフィールドのみを含めてください。
不明な場合は、指定のルールに従って空文字または適切なデフォルト値を返してください。
説明文や補足文を値に含めないでください。

**重要**: `scene_id` / `scene_order` はシステム側（GitHub）で付与するため、AI 出力に含めないでください。

---

- chapter: 物語上の章・大区分
  - 例: 「導入」「仲間集結」「対決」「帰還」「結び」
  - 物語全体の大きな流れが把握できるように整理する
  - 20字以内を目安
  - 必ず記述すること（空文字不可）

- scene_title: scene を一目で識別できる短い名称
  - 場面を端的に表す（20字以内を目安）
  - 人と後続AI が同じ scene をすぐ識別できる名前にすること
  - 例: 「桃が川から流れてくる」「仲間との出会い」「鬼ヶ島の決戦」
  - 必ず記述すること（空文字不可）

- scene_summary: scene 内で何が起こるかの要約
  - 1〜3文程度で場面の流れを説明する
  - target_age に応じた語彙・文体で記述する
  - 後工程が script 化しやすい粒度で要約すること
  - 必ず記述すること（空文字不可）

- scene_goal: その scene が物語上果たす役割
  - 1文程度で、この scene が物語全体の中でどんな役割を果たすかを記述する
  - 例: 「主人公の目的を明確にする」「仲間集結の開始を示す」
  - 構成妥当性・脚色方針・QC に使用する
  - 必ず記述すること（空文字不可）

- visual_focus: 映像上、何を主に見せるか
  - 具体的な名詞句または短文で記述する
  - STEP_06 Visual Bible・STEP_07 Image Prompts が参照するため、具体性を持たせること
  - 例: 「川を流れる大きな桃と驚くおばあさん」「協力して戦う桃太郎たち」
  - 必ず記述すること（空文字不可）

- emotion: scene の主感情・感情トーン
  - 短いテキスト（2〜4語程度）
  - Script トーン・演出・音調整・色調整に使用する
  - 例: 「ふしぎ、わくわく」「まじめ、やる気」「はらはら、すっきり」
  - 必ず記述すること（空文字不可）

- short_use: Short版で採用するか
  - 値は必ず `"Y"` または `"N"` のみ（他の値・説明文不可）
  - Y = Short版で採用する scene
  - N = Short版では省略してよい scene
  - Short版（約240秒）で物語の理解・感情曲線・満足感を保てる主要 scene に Y を付ける
  - 説明重複 scene や補助的 scene は N にしてよい
  - ただし、導入・転換点・山場・結びの欠落で物語が崩れないようにすること

- full_use: Full版で採用するか
  - 値は必ず `"Y"` または `"N"` のみ（他の値・説明文不可）
  - 初期実装では原則 `"Y"` を設定する

- est_duration_short: Short版での概算秒数（rough estimate）
  - 整数で記述
  - short_use = N の場合は 0 を設定する
  - short_use = Y の場合は Short版でのおおよその尺感を示す
  - あくまで rough estimate であり、最終的な尺調整は STEP_04_05_COMBINED で行う

- est_duration_full: Full版での概算秒数（rough estimate）
  - 整数で記述
  - scene_max_sec を超えてはならない
  - 全 scene の est_duration_full 合計が full_target_sec の ±15% 以内を目安とする（rough estimate）
  - 5秒以上

- difficult_words: 対象年齢に難しい語彙
  - 全角読点「、」区切りで列挙（例: 「どんぶらこ、家来、退治」）
  - 難語がない場合は空文字
  - 語彙平易化・字幕最適化に使用する

- easy_rewrite: difficult_words の言い換え候補
  - 全角読点「、」区切りで、difficult_words の順番に対応させる
  - 例: difficult_words が「どんぶらこ、家来」なら easy_rewrite は「ぷかぷか流れてくる、なかま」
  - difficult_words が空文字の場合は空文字
  - STEP_04_05_COMBINED でのやさしい表現化に使用する

- qa_seed: QA生成の種になる問い・観点
  - 短い問い/論点で記述する
  - STEP_09 Q&A Build で再利用されるため、後工程用途を意識した具体性を持たせる
  - 例: 「おばあさんは何を見つけたの？」「どうして勝てたの？」「みんなはどこへ向かったの？」
  - 必ず記述すること（空文字不可）

- continuity_note: 前後 scene との接続メモ
  - 短文で記述する
  - 持ち物・感情・構図・登場人物の連続性を支える情報を残す
  - Script / Visual / Edit Plan が参照するため、具体性を持たせること
  - 例: 「桃太郎の衣装・年齢感を固定する」「鬼ヶ島の外観を後続で統一する」
  - 必ず記述すること（空文字不可）
