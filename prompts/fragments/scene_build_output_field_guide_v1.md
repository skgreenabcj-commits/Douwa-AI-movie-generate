出力JSON の scenes 配列の各要素には、以下のフィールドのみを含めてください。
不明な場合は、指定のルールに従って空文字または適切なデフォルト値を返してください。
説明文や補足文を値に含めないでください。

- scene_id: scene 識別子
  - 形式: `SC-{project_num3桁}-{order2桁}` （例: SC-001-01, SC-001-02）
  - project_id が PJT-001 の場合は SC-001-XX となる
  - 必ず記述すること（空文字不可）

- scene_order: scene の順序
  - 1始まりの整数
  - scenes 配列内で重複してはならない
  - 昇順で並べること

- scene_title: scene タイトル
  - 場面を端的に表す（20字以内を目安）
  - 例: 「桃が流れてくる」「仲間との出会い」「鬼ヶ島の決戦」
  - 必ず記述すること（空文字不可）

- scene_summary: scene の内容サマリー
  - 2〜4文で場面の流れを説明する
  - target_age に応じた語彙・文体で記述する
  - 必ず記述すること（空文字不可）

- scene_purpose: scene の物語的役割・目的
  - 1〜2文で、この scene が物語全体の中でどんな役割を果たすかを記述する
  - 例: 「主人公の誕生と成長の始まりを描く導入場面」
  - 必ず記述すること（空文字不可）

- scene_type: scene 種別
  - 以下のいずれか1つを選ぶ（必ず列挙値から選択）:
    - `intro`: 物語の舞台・登場人物・状況を紹介する導入
    - `development`: 事件・出来事・旅・挑戦などが展開する場面（複数可）
    - `climax`: 物語の最大の緊張・対決・転換点（通常1〜2件）
    - `resolution`: クライマックスの結果・解決・和解が描かれる場面
    - `ending`: 物語の締めくくり・余韻・教訓が示される場面
  - 必ず記述すること

- scene_target_sec: この scene の推奨尺（秒）
  - 整数で記述（例: 30）
  - scene_max_sec を超えてはならない
  - 全 scene の合計が full_target_sec の ±15% 以内に収まること
  - 最小: 5秒

- key_characters: 登場する主要キャラクター
  - 全角読点「、」区切りで列挙（例: 「おじいさん、おばあさん、桃太郎」）
  - 登場キャラクターがいない場合は空文字

- key_events: scene 内で起こる主要な出来事
  - 1〜3文で記述する
  - 物語の流れとして重要な出来事を簡潔に記述すること
  - 必ず記述すること（空文字不可）

- visual_notes: 映像化の方向性ヒント
  - STEP_06 Visual Bible の制作者・AI が参照する映像イメージのヒント
  - 1〜2文で記述する（例: 「川辺の穏やかな風景。桃が流れてくるアニメーション。」）
  - 空文字でも可だが、可能な限り記述すること

- narration_style: ナレーション・語りのトーン
  - このscene のナレーションに求めるトーン・雰囲気を1文で記述する
  - 例: 「穏やかでゆっくりとした語り口。幼児が安心して聞けるトーン。」
  - 例: 「緊張感をもたせた語り口。わくわく感を煽るように。」
  - 必ず記述すること（空文字不可）
